# Capability Icons via FontAwesome — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add FontAwesome Free Solid icons to 33 of 41 SmartEVSE capability JSONs (32 new + 1 replacement on `mode`), leaving the standard `measure_rssi` and seven low-visibility diagnostic capabilities without custom icons.

**Architecture:** Per-capability SVG files live at `assets/icons/<capability_id>.svg` (one file per capability, even when the underlying FA glyph is reused). Each capability JSON in `.homeycompose/capabilities/` gains an `"icon"` field referencing its file. After all edits, `homey app validate` regenerates `app.json` from compose sources.

**Tech Stack:** Homey Apps SDK v3, `.homeycompose` source layout, Homey CLI 4.x (`homey app validate`), FontAwesome Free 6.x SVGs (acquired via `npm install --no-save @fortawesome/fontawesome-free`).

**Spec:** `docs/superpowers/specs/2026-04-26-capability-icons-fontawesome-design.md`

---

## File Structure

- `assets/icons/<capability_id>.svg` — 33 new SVG files, one per icon-bearing capability.
- `.homeycompose/capabilities/<capability_id>.json` — 33 files modified to add an `"icon"` field (1 replacement, 32 additions).
- `app.json` — regenerated by `homey app validate` from compose sources; not hand-edited.

The eight files NOT touched by this plan: `cp_pwm.json`, `cp_pwm_override.json`, `enable_c2.json`, `esp_uptime.json`, `ocpp.json`, `wifi_bssid.json`, `max_sum_mains_time.json`, `measure_rssi.json` (standard Homey capability — must keep its built-in icon).

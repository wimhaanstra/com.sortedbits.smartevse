# Flow Triggers: "Plug connected" and "Plug disconnected"

## Goal

Add two flow trigger cards that fire when the EV charger's `ev_plug_state`
capability transitions to `Connected` or `Disconnected`. Today the capability
exists and is used internally (by the `is_plugged_in` condition card and as
input to `deriveChargingState`), but there is no flow trigger because Homey
does not auto-generate triggers for custom enum capabilities.

This is **additive**: no existing flow card or capability is modified.

## Trigger cards

Both cards use the same minimal shape — only a `device` arg, no other args,
no tokens. The card title itself communicates the event.

| ID | Title EN | Title NL |
|---|---|---|
| `ev_plug_connected` | "Plug connected" | "Stekker aangesloten" |
| `ev_plug_disconnected` | "Plug disconnected" | "Stekker losgekoppeld" |

Both have `args: [{ name: "device", type: "device", filter: "driver_id=smartevse" }]`
and no `tokens` array.

## Behaviour

The triggers fire when the device's `ev_plug_state` MQTT value transitions to
a different value. The two valid MQTT values per `lib/types.ts` are
`Connected` and `Disconnected`.

Rules:

- **First observation after init** (`lastPlug === undefined`): do not fire.
  Avoids a spurious trigger on app startup or device pairing.
- **Same value as before**: do not fire (handles duplicate MQTT messages).
- **Real transition to `Connected`**: fire `ev_plug_connected`.
- **Real transition to `Disconnected`**: fire `ev_plug_disconnected`.
- **Any other value** (defensive): do not fire either trigger. The
  `PlugState` type only allows the two values above; this branch exists only
  so an unexpected MQTT payload doesn't fire `ev_plug_disconnected` by
  accident via an `else` fall-through.

### `drivers/smartevse/device.ts`

Inside `onTopic()`, replace the existing one-liner:

```typescript
if (res.capId === 'ev_plug_state') this.lastPlug = res.value as PlugState;
```

with the transition-detection block:

```typescript
if (res.capId === 'ev_plug_state') {
  const next = res.value as PlugState;
  if (this.lastPlug !== undefined && this.lastPlug !== next) {
    if (next === 'Connected') this.tryTrigger('ev_plug_connected', {});
    else if (next === 'Disconnected') this.tryTrigger('ev_plug_disconnected', {});
  }
  this.lastPlug = next;
}
```

No new fields, no helper extraction, no driver.ts change. The existing
`tryTrigger` helper (already extended in the prior feature to accept an
optional `state` payload) is reused with empty tokens `{}`.

Auto-regenerated by homeycompose. Not hand-edited.

## Testing

No new unit tests. The transition-detection logic is two `if` checks with
literal string comparisons — extracting a helper would create more code than
it removes. Validation is via `npm run build` + `npm run lint` + the existing
`npm test` suite (must remain at 88/88).
